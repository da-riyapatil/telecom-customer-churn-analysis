# Project Summary

## Overview
The **Telecom Customer Churn Analysis Dashboard** is a Power BI project focused on understanding customer churn in a telecom business.

This was a **Forage internship-style project**, where the dataset and business problem were already provided. My role was to analyze the data, prepare it for reporting, build DAX measures, and create a dashboard to highlight churn patterns and customer behavior.

## Objective
The main objective of this project was to identify the factors linked with higher customer churn and present them in a clear and interactive dashboard.

The analysis was designed to help answer questions such as:
- Which customers are more likely to churn?
- Does contract type affect churn?
- Which payment methods show higher churn?
- Are support-related services linked with retention?
- Does churn happen more in the early customer lifecycle?

## Dataset
The dataset contains **7,043 customer records** with fields related to:
- customer demographics
- contract details
- billing information
- internet and support services
- tenure
- churn status

One important data preparation step was handling blank values in **TotalCharges** and converting that field into numeric format for analysis.

## Work Completed
In this project, I worked on:
- cleaning and preparing the dataset in Power BI
- checking missing and blank values
- creating calculated columns for customer segmentation and tenure-based analysis
- building DAX measures such as **Total Customers**, **Churn Rate**, **Revenue**, and other KPI metrics
- designing a one-page interactive dashboard
- analyzing churn trends across customer and service categories

## Dashboard Focus Areas
The dashboard was built to explore churn across key business dimensions, including:
- overall churn rate
- contract type
- internet service type
- payment method
- support services
- monthly charges
- tenure behavior

## Key Outcome
The dashboard showed that churn was especially higher among:
- **month-to-month contract customers**
- **fiber optic users**
- customers using **electronic check**
- customers without **tech support** or **online security**
- customers in their **early tenure period**
